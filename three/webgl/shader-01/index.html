# Three.js Shader 사용하기 (1)

Three.js는 WebGL을 간편하게 사용하게 해주는 도구입니다. 하지만 커스텀한 기능을 요구할 때 Shader을 사용해야할 때가 있습니다.
아래 튜토리얼에서 Three.js 에서 Shader를 어떻게 사용하는지 알아보겠습니다.

an-introduction-to-shaders-part-1 (opens new window)

Shader 는 아래 두 가지 종류가 있습니다.

  • VertexShader
  • FragmentShader

# VertexShader

  • 버텍스셰이더는 주로 물체의 정점 정보에 수학적인 연산을 함으로써 물체에 특별한 효과를 주는 데 쓰입니다.
  • 버텍스셰이더는 반드시 gl_Position을 정의해야 합니다.
  • gl_Position은 정점을 화면상의 위치로 지정하는 4D float Vector 입니다.
  • 실제로 우리는 3D 위치(x, y, z)를 2D 화면에 투영하고 있습니다.
  • Three.js 를 이용한다면 gl_Position을 간단하게 설정할 수 있습니다.

# FragmentShader

  • 정점들이 2D 화면에 투영됬다면 사용할 색상, 텍스처, 조명도 필요합니다.
  • 프레그먼트셰이더는 위 작업을 수행할 수 있는 4D float Vector 입니다.
  • 프레그먼트셰이더는 반드시 gl_FragColor 변수를 설정하거나 버려야 합니다.
  • 만약 삼각형을 형성하는 세 개의 정점이 존재하고, 그 삼각형 내의 픽셀을 그리기 위해 각 픽셀에 대한 데이터를 나타내는 것이 프레그먼트 입니다.
  • 프레그먼트는 구성 정점에서 보간된 값을 가집니다. (빨간색 정점의 이웃이 파란색이라면 빨강,보라,파랑으로 보간되어 색상이 나타납니다.)

# Shader Variables

  • Uniforms: 버텍스셰이더와 프래그먼트셰이더로 보내지며 모든 프레임에 동일한 값을 가지는 변수입니다. (Ex. 빛의 위치)
  • Attributes: 각 정점에 적용되는 값으로 버텍스셰이더에서만 사용 가능합니다. 각 정점과 일대일 관계를 갖습니다.
  • Varyings: 버텍스셰이더에서 선언된 변수로, 프로그래먼트셰이더와 공유합니다. 이를 위해 버텍스셰이더와 프래그먼트셰이더에서 동일한 형식과 이름의 가변 변수로 선언해야 합니다. (가장 대표적인 예시는 법선입니다. 법선은 조명 계산에 사용될 수 있기 때문입니다.)

# 구현

let vertex = `
    void main() {
        gl_Position = projectionMatrix * modelViewMatrix  * vec4(position,1.0);
    }
`;

let frag = `
    void main() {
        gl_FragColor = vec4(0.0, 0.5, 1.0, 1.0); // R, G, B, A
    }
`;

var geometry = new THREE.BoxGeometry( 20, 20, 20 );
var material = new THREE.ShaderMaterial({
    vertexShader : vertex,
    fragmentShader : frag
});

var cube = new THREE.Mesh( geometry, material );
threeLayer.scene.add( cube );
